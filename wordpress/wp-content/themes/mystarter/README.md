# MyStarter Theme

Starter WordPress child theme based on Twenty Twenty-Four with a curated Gutenberg experience and a custom library of blocks designed for marketing pages.

## Installation

1. From the theme directory run `npm install`.
2. Build assets with `npm run build` (use `npm run start` for watch mode).
3. Activate the **MyStarter** theme in `Apparence → Thèmes`.
4. Flush permalinks after activation to expose the `Actualités` custom post type.

## Available npm scripts

- `npm run start` – development build with file watching.
- `npm run build` – production build of scripts and styles.
- `npm run lint:js` – lint JavaScript/JSX files.
- `npm run lint:css` – lint styles.
- `npm run format` – run Prettier on supported files.

## Custom blocks

All native blocks are disabled; the builder exposes the following blocks instead:

- `Image` & `Image pleine largeur`
- `Galerie`
- `Slider`
- `Texte + image`
- `Groupe de boutons`
- `Citation`
- `Call to action`
- `Bloc rebond`
- `Remontée actualités`
- `Hero banner` (slider, image ou vidéo)

They live under the “MyStarter” block category and include both editor/front styles.

## Additional features

- `theme.json` with curated color palette, typography, layout and spacing scales.
- `Actualités` custom post type for news, surfaced inside the custom blocks.
- Block patterns (hero & CTA) registered via the `patterns/` directory.
- Optional slider behaviour handled via `src/frontend/index.js`.
- Build tooling powered by `@wordpress/scripts`, Sass, PostCSS, ESLint, Stylelint and Prettier.

## Folder structure

- `src/` – source files for blocks, styles and front-end scripts.
- `build/` – compiled assets generated by the build step.
- `patterns/` – PHP registration files for block patterns.

## Development workflow

1. Edit files under `src/`.
2. Run `npm run start` during development for live rebuilds.
3. Commit generated `build/` assets when distributing the theme.

## Cleaning up

If you deactivate the theme, no additional cleanup is required. To remove Node dependencies run `rm -rf node_modules package-lock.json`.
